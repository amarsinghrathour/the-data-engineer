# Notebooks

Lessons are grouped by phase (`00_foundations` … `09_capstone`). Follow the **notebook standard** below so every file stays consistent for self-study and instructors.

## Notebook standard (use in every notebook)

Each notebook should include these sections **in order** (use Markdown headings):

1. **Concept explanation** — what you are learning and why it matters in production.
2. **Real-world analogy** — one concrete analogy that maps to data systems.
3. **Code examples** — minimal, runnable cells with clear inputs/outputs.
4. **Practice problems** — short exercises without full solutions first (add a collapsed solution optional).
5. **Assignment** — one deliverable that can be checked (artifact path, test, or rubric).
6. **Interview questions** — 3–5 prompts tied to this notebook’s topic.
7. **Your tasks** (required) — a final `## Your tasks` section: a **checklist** (`- [ ] …`) of everything the learner should do **later** (re-run, practice, assignment, bonus). This is the handoff so self-study students never close a lesson without a concrete to-do list. You can repeat practice/assignment bullets here as checkboxes; add time estimates if useful.

Optional but encouraged: **failure modes** (what breaks at scale) and **references** (docs, papers, official guides).
