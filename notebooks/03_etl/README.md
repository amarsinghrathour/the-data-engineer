# Phase 3 — ETL pipelines

**Weeks 9–11**

This is where you move from **“I know tools”** to **“I build systems”**: extract → transform → load, data flow, and how real pipelines break (and get fixed).

**Project:** end-to-end ETL system (see `05_etl_project.ipynb`).

## Notebooks (in order)

| File | Topic |
|------|--------|
| `01_etl_fundamentals.ipynb` | ETL vs ELT, minimal extract/transform/load |
| `02_data_cleaning_and_transformation.ipynb` | Missing values, types, dedupe, pipeline |
| `03_building_etl_pipeline.ipynb` | API → transform → PostgreSQL + logging |
| `04_incremental_pipeline.ipynb` | Watermarks, state table, skip reprocess |
| `05_etl_project.ipynb` | E-commerce ETL capstone (API + SQL analytics) |

**Prerequisites:** Phase 1 (Python + APIs + logging) and Phase 2 (SQL) help; not strictly blocking for notebook 1.

Follow [`../README.md`](../README.md) — every notebook ends with **Your tasks**.
